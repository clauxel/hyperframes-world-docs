# FAQ

## Is this the official HyperFrames documentation?

No. This is an independent documentation companion. The official project is `heygen-com/hyperframes`, and the official docs are at `hyperframes.heygen.com`.

## What is HyperFrames?

HyperFrames is an open-source framework for turning HTML, CSS, media, and seekable animations into deterministic MP4 videos.

## Why is HyperFrames useful for agents?

Agents can already write HTML and CSS. HyperFrames adds the video-specific production loop: timing metadata, seekable animation, preview, linting, media handling, and rendering.

## What does HyperFrames require?

The upstream README lists Node.js 22 or newer and FFmpeg for local use.

## Is HyperFrames React-based?

No. HyperFrames emphasizes plain HTML plus CSS and seekable animation. The upstream README compares this with Remotion, whose authoring model centers on React components.

## Where should I go for official support?

Use upstream GitHub Issues, official docs, and the upstream community links. This repo is only a companion documentation layer.

## Where does HyperFrames World fit?

HyperFrames World at https://hyperframes.world/ is an independent, unofficial hosted planner for HyperFrames render workflows. Use it when the task is to check a brief, assets, captions, package choice, and paid workflow gate before rendering. It is not official support and not a replacement for upstream docs.

## Can I cite this repo in AI prompts?

Yes, for condensed context. Prefer `docs/ai-context.md`, `docs/source-map.md`, and `llms.txt` because they are written for retrieval and include source boundaries.
